## 一、模块 12：记账系统集成（Books Integration）

### 1.1 为什么必须补这一步

原规划的数据流是「OCR 识别 → GIFI 映射 → T2 报税」，识别出的交易只服务于年度报税，**没有形成客户的正式账本**。这有两个问题：

- T2 的 Schedule 100/125 需要**期末试算表**（Trial Balance），而试算表来自日常记账。没有记账环节，报税数据就成了无源之水，只能靠客户另行提供会计软件导出，流程断裂。
- Dext 的核心商业价值恰恰是「识别后自动推送到 Xero/QBO 记账」。我们只做识别不做录入，等于只复刻了 Dext 的一半，客户仍需手工录入。

补上这一步后，完整数据流为：

```
客户上传文件
    ↓
Veryfi OCR 识别（模块 2）
    ↓
双重科目映射（模块 3 扩展）
    ├── GL 科目映射 → 推送记账系统（模块 12，本节）
    └── GIFI 映射 → T2 预填（模块 4）
    ↓
期末：记账系统试算表 → GIFI 汇总 → Schedule 100/125
    ↓
会计师复核（模块 5）→ TaxCycle → CRA（模块 6）
```

### 1.2 两条录入路径

**路径 1：客户已使用 QuickBooks Online / Xero（主路径）**

| 环节 | 实现方式 |
|------|---------|
| 授权连接 | OAuth 2.0，客户在门户内一键授权自己的 QBO / Xero 账户 |
| 读取科目表 | 通过 API 拉取客户的 Chart of Accounts（GL 科目） |
| 双重映射 | 每条交易同时映射 GL 科目（用于记账）和 GIFI 代码（用于报税），系统维护 GL ↔ GIFI 对照表，一次确认两处生效 |
| 写入交易 | QBO API：创建 Expense / Bill / Journal Entry；Xero API：创建 Bank Transaction / Bill |
| 去重对账 | 推送前与 QBO/Xero 已有交易（含银行 feed 自动导入的）按「日期 + 金额 + 供应商」模糊匹配，命中则标记为疑似重复，交人工确认，防止重复入账 |
| 附件回传 | 原始收据图片作为附件挂到 QBO/Xero 交易上（审计留痕） |
| 状态追踪 | 每条交易显示：待推送 / 已推送 / 推送失败 / 疑似重复待确认 |

**API 成本与准入**：QBO 与 Xero 的 API 本身免费。QBO 生产环境密钥需通过 Intuit 安全审查（约 2–4 周）；Xero 需完成 App Partner 认证。两者均无按调用量收费。

**路径 2：客户没有会计软件（内置轻量总账 Simple Ledger）**

面向最小微的企业（大量 T2 客户实际没有 QBO 订阅）：

- 系统内置一套以 GIFI 结构为默认科目表的轻量总账
- OCR 识别的交易经确认后直接入账，期末自动生成试算表，无缝进入 T2 预填
- 支持导出标准格式，客户后期升级到 QBO/Xero 可迁移
- 这条路径让「无会计软件的小企业」成为可服务对象，扩大了客群下限

### 1.3 开发排期建议

模块 12 归入 **Phase 2**（与 GIFI 映射引擎同期开发，因为共享映射基础设施）：

- Phase 2 前半：轻量总账（路径 2）+ GL↔GIFI 对照表 —— 无外部依赖，先跑通闭环
- Phase 2 后半：QBO 集成（路径 1，加拿大市占率最高）
- Phase 3：Xero 集成（与 TaxCycle 对接同期，Xero 与 TaxCycle 同属一家公司，生态协同）

## 二、模块 13 / 方向 C：AI 人才基地（人工兜底体系）

### 2.1 定位

合伙人正在筹备的 AI 人才基地，本质是**为 AI 工具提供人工兜底（Human-in-the-Loop）的运营团队**。它与本系统的关系是双向的：

- 对内：承接本系统所有低置信度任务的人工处理，让产品敢于承诺「100% 结果经人复核」
- 对外：以本系统为第一个标杆客户，向其他 AI 工具输出「按件计费的人工复核服务」，独立创收

### 2.2 与本系统的接口设计

所有 AI 环节产生的低置信度结果，自动生成工单推入人才基地任务队列：

| 触发场景 | 工单类型 | 人工操作 |
|---------|---------|---------|
| OCR 置信度低于阈值 | 单据校验 | 对照原图修正金额/日期/供应商 |
| GIFI/GL 映射置信度「低」 | 科目判定 | 选定正确科目，结果写回规则库 |
| 金额异常（超历史均值 3 倍） | 异常复查 | 确认真实性，标注原因 |
| 疑似重复交易 | 去重确认 | 判定保留/合并 |
| 客户材料缺失判断 | 清单核对 | 确认缺什么、生成催件话术 |

**工作台形态**：任务列表 → 左侧原始文件 + 右侧 AI 建议 → 确认/修正 → 提交。每单处理结果自动回流为训练/规则数据，形成**数据飞轮**：人工修正越多 → 映射规则库越准 → 需要人工的比例越低 → 单位成本持续下降。

**SLA 分级**：普通件 24 小时内、加急件 4 小时内（报税季截止日前自动升级为加急）。

### 2.3 计费与成本模型

- 内部结转价（本系统使用）：**约 CA$0.3–0.8/工单**（按工单类型定价，科目判定简单、单据校验较贵）
- 对外报价（服务其他 AI 工具）：CA$0.8–2.0/工单 或坐席包月 CA$2,500–3,500/人/月
- 按规划的置信度分布（70–80% 自动通过），500 家企业 × 100 交易/月 × 25% 需人工 ≈ 12,500 工单/月，内部成本约 **CA$5,000–8,000/月**——这笔成本已计入第三节运营成本模型的「规模档」

### 2.4 战略意义

1. **风险转卖点**：AI 报税最大的顾虑是错报责任。有人才基地兜底，「AI 提效 + 人工把关」成为可宣传的服务承诺，而非藏在免责条款里的风险。
2. **第三条业务曲线**：SaaS（方向 A）、加盟（方向 B）之外，人才基地（方向 C）可独立对外接单（数据标注、AI 结果审核、内容审核），不依赖本系统存活。
3. **加盟体系的支撑**：Phase 4 加盟商上线后，加盟商的复核工作量可选择「自己做」或「付费转给人才基地」，人才基地成为联盟的中央处理厂。

## 三、开发周期与现金需求（修订 v2：全合作路线）

> 币种：CAD（标注 USD 处除外）。
>
> **路线定调：全部走合作，不自建认证。** OCR 接 Veryfi、记账接 QBO/Xero 官方 API、CRA 提交经 TaxCycle（会计师在桌面端完成最后一步点击）。**不申请 CRA 自有 EFILE 认证**（列为远期可选项，不进本计划和预算）；SOC 2 Type II 推迟到有机构客户强制要求时再评估。砍掉这两项后，周期和现金需求较上一版大幅下降。

### 3.1 里程碑排期

| 里程碑 | 内容 | 自筹版（技术合伙人 + AI） | 加速版（+1 名合同开发） |
|--------|------|--------------------------|------------------------|
| M1：门户可演示 | 客户门户 + Veryfi OCR + 基础报告 | 第 3 个月 | 第 2 个月 |
| M2：核心闭环 | 双重科目映射 + QBO/轻量总账 + 复核工作台 | 第 8 个月 | 第 4–5 个月 |
| M3：报税链路通 | GIFI CSV 导出 → TaxCycle 导入验证 | 第 9–10 个月 | 第 5–6 个月 |
| M4：开始收费 | 3–5 家种子事务所跑完整流程，订阅收费 | 第 10–12 个月 | **第 6 个月** |

两点说明：

- **GIFI CSV 导出本身只是约 2 周的开发量**——TaxCycle 官方支持手动导入 `.CSV/.GFI` 是标准功能。TaxCycle API 协议（自动导入）从第 2 个月起并行申请谈判，**谈不下来也不阻塞**，只是会计师多一步手动导入。
- 外部依赖只剩两个轻量项：Intuit 生产密钥审查（2–4 周，Phase 2 期间并行）和 TaxCycle 谈判（非阻塞）。没有任何一年级别的认证周期。

### 3.2 现金需求明细（逐项）

**自筹版（12 个月到可收费）：**

| # | 项目 | 计算依据 | 金额 |
|---|------|---------|------|
| 1 | UI/UX 设计外包 | 客户门户 + 复核工作台两套界面，一次性 | CA$5k–8k |
| 2 | 云资源（AWS ca-central-1） | 约 CA$60/月 × 12 个月 | ~CA$0.7k |
| 3 | Veryfi OCR | 开发期用 100 份/月永久免费额度 | $0 |
| 4 | Claude API 调试与评测 | 约 CA$50/月 × 9 个月（Phase 2 起） | ~CA$0.5k |
| 5 | 邮件/短信/域名/工具 SaaS | 约 CA$80/月 × 12 个月 | ~CA$1k |
| 6 | GIFI 规则库 + 测试数据整理 | 外包人才基地，约 200 小时 | CA$3k–5k |
| 7 | TaxCycle 授权（验证导入用） | 1 席 × 1 年 | ~CA$1.1k |
| 8 | E&O 职业责任保险 | 首年，上线前投保 | CA$3k–5k |
| 9 | 法务杂项 | 服务条款、隐私政策、数据处理协议（模板 + 律师审阅） | CA$2k–4k |
| | **合计** | | **约 CA$16k–25k** |

**加速版（6 个月到可收费）**——自筹版第 2–5 项按 6 个月折算（约省 CA$1k），另增：

| # | 项目 | 计算依据 | 金额 |
|---|------|---------|------|
| 10 | 合同制全栈开发 | 1 名 × 5 个月 × CA$8k–10k/月（承担门户/集成，合伙人聚焦 AI 引擎） | CA$40k–50k |
| 11 | 设计加急 + 基础品牌 | 压缩交付周期溢价 | CA$2k–3k |
| | **加速版合计** | | **约 CA$56k–74k** |

**明确不花的钱（与上一版的差异来源）：**

| 砍掉/推迟项 | 上一版预算 | 处理 |
|------------|-----------|------|
| CRA 自有 EFILE 认证（法务 + 流程） | CA$10k–20k | ❌ 不做，长期走 TaxCycle 合作 |
| SOC 2 Type II 审计 | CA$25k–40k + 年审 | ⏸️ 推迟，机构客户强制要求时再评估 |
| 渗透测试 | CA$8k–15k | ⏸️ 推迟，先用自动化安全扫描 |

**Phase 4 加盟启动预算（独立决策，不含在上述数字内）**：加盟法律文件（安省《Arthur Wishart 法》披露文件）+ 品牌 VI 约 CA$25k–50k，启动时点取决于 SaaS 验证结果。

### 3.3 结论

| | 自筹版 | 加速版 |
|---|--------|--------|
| 到可收费 | 12 个月 / **CA$16k–25k** | 6 个月 / **CA$56k–74k** |

加速版多花的约 CA$40k–50k 全部是合同开发人力，买到的是提前半年收费 + 多抢一个报税季（2–4 月）。这个量级的现金缺口，政府扶持资金（见第五节）基本可以覆盖，股权融资从「必需」变成「可选加速器」。

## 四、运营成本模型

### 4.1 月度变动成本（按业务规模三档）

| 成本项 | 起步档（≤20 家企业） | 成长档（~200 家） | 规模档（~500 家） |
|--------|--------------------|------------------|------------------|
| Veryfi OCR | $0（免费 100 份/月额度） | ~US$500（最低消费） | ~US$650 |
| Claude API（GIFI/GL 映射） | ~US$10 | ~US$80 | ~US$200 |
| AWS ca-central-1（应用 + RDS + S3） | ~CA$50 | ~CA$250 | ~CA$500 |
| 邮件/短信提醒（SES + Twilio） | ~CA$10 | ~CA$60 | ~CA$150 |
| 域名、监控、杂项 SaaS | ~CA$30 | ~CA$50 | ~CA$80 |
| 人工兜底（人才基地内部结转，25% 交易需人工） | ~CA$150 | ~CA$2,500 | ~CA$5k–8k |
| **月合计（USD 按 1.35 折算）** | **~CA$250** | **~CA$3.7k** | **~CA$7k–10k** |

> Claude API 估算依据：500 家 × 100 交易/月 = 5 万次映射，每次约 800 输入 + 100 输出 token，按 Sonnet 价格 ≈ US$200/月；若高置信度部分改用 Haiku 可再降一半以上。

### 4.2 年度固定成本

| 项目 | 金额 | 起始阶段 |
|------|------|---------|
| E&O 职业责任保险 | CA$3k–8k/年 | Phase 1 上线即需 |
| SOC 2 Type II 年审 | CA$15k–30k/年 | ⏸️ 已推迟——仅当机构客户强制要求时启动 |
| TaxCycle 授权 | CA$1,102/用户/年 | 多数场景由会计师自购，总部自用 1–3 席 |
| 法务/会计杂项 | ~CA$5k/年 | 全程 |

### 4.3 盈亏平衡测算

规模化前的关键节点——**成长档**（200 家企业，约 10–15 家事务所客户）：

- 月成本：变动 ~CA$3.7k + 固定分摊 ~CA$1.5k ≈ **CA$5.2k/月**
- 收入：按专业版 $399/月 计，**14 家事务所订阅即打平**；若其中 3 家为机构版（$799），10 家即打平
- 加盟收入（Phase 4 起）为额外弹性：每签 1 家加盟商 = CA$15k–25k 一次性 + CA$2.4k–4.8k/年

## 五、政府扶持资金与融资路径

> 加拿大对科技型中小企业的扶持体系相当完整，本项目（CCPC + AI 研发 + 雇佣计划）与其中几项高度匹配。金额与规则以申请时官网为准（政策每年调整，下表基于 2026 年初信息整理）。

### 5.1 可申请项目清单（按匹配度排序）

| 项目 | 性质 | 金额量级 | 与本项目匹配度 |
|------|------|---------|---------------|
| **SR&ED 科研税收抵免**（联邦 + 安省叠加） | 退税型税收抵免 | 合格研发支出约 35%（联邦，CCPC 可退现金）+ 安省约 8–10% 叠加，**研发工资综合回收约 40%+** | ⭐⭐⭐⭐⭐ AI 科目映射引擎、OCR 管道调优属于「实验性开发」，是教科书式的合格项目；2024 年底政策还上调了支出上限 |
| **NRC IRAP 工业研究援助** | 无偿补贴（不占股） | 首个项目通常 CA$50k–150k，可覆盖研发工资最高 80%、合同支出 50%，**按月报销** | ⭐⭐⭐⭐⭐ 要求：注册公司、<500 人、有增长计划。配备免费技术顾问（ITA）。对加速版现金流帮助最大，应第 1 个月就联系 |
| **SWPP 学生实习工资补贴** | 工资补贴 | 每名 co-op 学生最高约 CA$7k/学期 | ⭐⭐⭐⭐ 直接补贴**人才基地**的用人成本——复核坐席用 co-op 学生起步 |
| **CSBFP 政府担保小企业贷款** | 贷款（政府担保 85%） | 最高 CA$1.15M（营运资金类上限 CA$150k），经银行申请 | ⭐⭐⭐ 债权型补充，不稀释股权 |
| **BDC 创业融资** | 贷款 | 创业贷款至约 CA$250k，另有科技企业专项 | ⭐⭐⭐ 国有开发银行，对无抵押科技企业相对友好 |
| **Futurpreneur** | 贷款 + 导师 | 至约 CA$75k（含 BDC 配套） | ⭐⭐⭐ 仅限 18–39 岁创始人，视合伙人年龄 |
| **Mitacs Accelerate** | 研发实习配比资助 | CA$15k/项目，企业只出一半 | ⭐⭐⭐ 「GIFI 映射准确率」可与高校实验室立项 |
| **安省省级项目** | 小额补贴 + 免费辅导 | Starter Company Plus 约 CA$5k 等 | ⭐⭐ 量小但门槛低 |
| **孵化器/加速器**（DMZ、MaRS、Communitech） | 资源 + 投资人网络 | 不直接给钱 | ⭐⭐⭐ 通往天使/VC 的主要通道 |

⚠️ **CDAP（数字化采纳计划）已于 2024 年关闭**，网上旧资料仍常提及，勿列入计划。

### 5.2 资金组合策略

- **第一年可回收/获得约 CA$50k–150k**（IRAP + SR&ED 退税 + 工资补贴合计），**已能覆盖加速版全部现金需求（CA$56k–74k）**——理论上「政府资金 + 少量合伙人出资」即可跑完，不融资也走得通
- 时间差要注意：**IRAP 是事前立项、按月报销**（对现金流最友好，正好对冲合同开发的月度工资）；**SR&ED 是事后退税**（随税务年度申报后数月到账，第一年帮助滞后但金额最大）
- 本项目天然优势：我们自己就是做税务申报的——申报 SR&ED、辅导加盟商申领扶持资金，既是内部能力也可以变成对客户的增值服务
- **股权融资角度**：不再是必需项，而是可选加速器——若融资，建议目标 pre-seed **CA$100k–150k**，用于加速版预算 + 首年运营 + 种子客户获客；taxtech 赛道资本活跃（调研报告中 Sphere 获 a16z 领投 US$21M A 轮），「AI + 强制电子申报政策红利 + 加盟网络」的故事对种子投资人有吸引力，且预算小意味着稀释低

### 5.3 申请优先级（建议顺序）

1. **立即**：联系 NRC IRAP 分配 ITA 顾问（免费，立项谈判周期 1–2 个月）
2. **立即**：找 SR&ED 顾问做预评估（多数按退税额比例收费，无前期成本）
3. **第 1–2 月**：注册 SWPP 雇主资格，为人才基地招 co-op 学生
4. **第 2–3 月**：进入一家孵化器（DMZ/MaRS），铺天使/种子轮网络
5. **按需**：CSBFP/BDC 贷款作为后备额度

|---------|------|
| 模块 3（GIFI 映射） | 升级为**双重映射**：GL 科目（记账用）+ GIFI（报税用），共享置信度体系和学习机制 |
| 模块清单 | 新增模块 12（记账系统集成）、模块 13（AI 人才基地接口） |
| 业务方向 | 「双轮驱动」升级为**三轮驱动**：A SaaS + B 加盟 + C 人才基地 |
| Phase 2 范围 | 增加轻量总账 + QBO 集成 |
| Phase 3 范围 | 增加 Xero 集成 |
| 技术路线 | 定调**全合作路线**：Veryfi + QBO/Xero + TaxCycle 全部走对接；❌ 不申请 CRA 自有 EFILE 认证；⏸️ SOC 2 与渗透测试推迟 |
| 排期 | 修订为**自筹 12 个月 / 加速 6 个月到可收费**；无任何年级别认证周期，唯一外部依赖（TaxCycle 谈判）非阻塞 |
| 现金需求 | 逐项列明：自筹版 **CA$16k–25k**，加速版 **CA$56k–74k**（差额为合同开发人力） |
| 资金来源 | 新增政府扶持资金章节（SR&ED、IRAP、SWPP 等），第一年可回收约 CA$50k–150k，可覆盖加速版全部现金需求 |
| 风险表 | 「AI 映射错误」的应对由“会计师复核”强化为“会计师复核 + 人才基地兜底双保险” |
