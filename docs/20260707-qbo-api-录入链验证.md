# QuickBooks Online API — 录入链技术验证

> 日期：2026-07-07
> 目的：验证 MVP 核心链的终点「一键录入 QuickBooks Online」在技术上通不通、有哪些坑。
> **诚实声明**：本文档是**文档级验证**（基于 Intuit 官方 API v3 整理），**尚未经沙箱实跑**。
> 真正跑通需要你的 Intuit 开发者凭据 + sandbox company；第 7 节附**可运行脚本**，凭据到位即可实测。
> 易变项（minorversion 号、税码、费率）已逐处标注「待沙箱实测确认」。

---

## 0. 结论先行

**录入链技术上可行，无阻塞性障碍。** 五个动作（读科目表 / 查建供应商 / 建 Bill / 挂附件 / 去重）都有明确 API 支持。三个必须提前知道的坑：

1. ⚠️ **加拿大销售税**：Intuit sandbox 默认是**美国公司**，没有 GST/HST 税码。要验证加拿大税逻辑，需要加拿大区测试公司；建 Bill 时加拿大公司要处理 `TxnTaxDetail` / `TaxCodeRef` / `GlobalTaxCalculation`，美国逻辑不能直接搬。
2. ⚠️ **去重要自己做**：QBO 没有原生「重复检测」API，需在我们系统侧建指纹（供应商+发票号+金额+日期），QBO 侧用 query 二次校验。
3. ⚠️ **生产准入有周期**：生产密钥要过 Intuit 安全审查（约 2–4 周）。MVP 全程用 sandbox 开发，并行提交审查。

## 1. 认证与环境

| 项 | 值 |
|----|-----|
| 授权模式 | OAuth 2.0 Authorization Code |
| Scope | `com.intuit.quickbooks.accounting` |
| Authorize 端点 | `https://appcenter.intuit.com/connect/oauth2` |
| Token 端点 | `https://oauth.platform.intuit.com/oauth2/v1/tokens/bearer` |
| Access token | 有效期 1 小时；用 refresh token 续（refresh token 每次刷新轮换，约 100 天有效） |
| **Sandbox base URL** | `https://sandbox-quickbooks.api.intuit.com` |
| **Production base URL** | `https://quickbooks.api.intuit.com` |
| API 路径 | `/v3/company/{realmId}/{resource}` |
| realmId | 授权成功回调时返回的公司 ID，标识写哪个客户的账套 |
| minorversion | 查询参数 `?minorversion=NN`，用当前最新（待沙箱实测确认当前号，约 73–75） |
| 限流 | 约 500 请求/分钟/公司；超限返回 429/403，需指数退避 + 尊重 `Retry-After` |
| 数据格式 | 请求/响应默认 JSON（`Accept: application/json`） |

> 多租户对应关系：**一个客户企业 = 一个 realmId**。会计师帮 N 个客户 = 我们存 N 组 (realmId + refresh token)。这正好落实需求文档里「schema 从一开始带 client_id」。

## 2. 录入链的五个 API 动作

### 动作 1：读客户科目表（分类的下拉来源）

```
GET /v3/company/{realmId}/query?query=SELECT * FROM Account WHERE AccountType = 'Expense'&minorversion=NN
```
返回每个科目的 `Id / Name / AccountType / AccountSubType / FullyQualifiedName`。
用途：AI 分类的目标集合就是这张表；`Account.Id` 即建 Bill 时 `AccountRef.value`。

### 动作 2：查 / 建供应商（VendorRef 的来源）

```
GET  /v3/company/{realmId}/query?query=SELECT * FROM Vendor WHERE DisplayName = 'Tim Hortons'
POST /v3/company/{realmId}/vendor        Body: { "DisplayName": "Tim Hortons" }   # 查不到才建
```
拿到 `Vendor.Id` → 建 Bill 的 `VendorRef.value`。

### 动作 3：创建 Bill（录入的核心）

```
POST /v3/company/{realmId}/bill?minorversion=NN
```
**最小必填 JSON**（美国公司）：
```json
{
  "VendorRef": { "value": "56" },
  "Line": [
    {
      "DetailType": "AccountBasedExpenseLineDetail",
      "Amount": 100.00,
      "AccountBasedExpenseLineDetail": {
        "AccountRef": { "value": "7" }
      }
    }
  ]
}
```
常用可选：`TxnDate`（账单日）、`DueDate`（到期日）、`DocNumber`（发票号，去重关键）、`CurrencyRef`、`PrivateNote`、`APAccountRef`。

**加拿大公司加税**（待沙箱实测确认字段组合）：
```jsonc
{
  "VendorRef": { "value": "56" },
  "GlobalTaxCalculation": "TaxExcluded",
  "Line": [{
    "DetailType": "AccountBasedExpenseLineDetail",
    "Amount": 100.00,
    "AccountBasedExpenseLineDetail": {
      "AccountRef": { "value": "7" },
      "TaxCodeRef": { "value": "HST ON" }   // 税码 Id 因公司而异
    }
  }],
  "TxnTaxDetail": { /* 税额明细，或让 QBO 按 TaxCodeRef 自动算 */ }
}
```

### 动作 4：挂原始收据附件（审计留痕）

Bill 实体本身没有附件字段，用 **Attachable** 实体：
```
POST /v3/company/{realmId}/upload      Content-Type: multipart/form-data
  part 1  file_metadata_01 = JSON: {
            "FileName": "invoice.pdf", "ContentType": "application/pdf",
            "AttachableRef": [{ "EntityRef": { "type": "Bill", "value": "<新建Bill的Id>" } }]
          }
  part 2  file_content_01 = <二进制文件>
```
顺序：先建 Bill 拿到 Id → 再 upload 把原件关联上去。

### 动作 5：去重（防重复入账）

QBO 无原生查重。两层防线：
1. **系统侧指纹**（主）：`供应商 + 发票号(DocNumber) + 金额 + 日期` 做唯一指纹，入库前先查我们自己的表。
2. **QBO 侧二次校验**：`SELECT * FROM Bill WHERE DocNumber = 'INV-123'`（注意 QBO 默认不强制 DocNumber 唯一，只作辅助）。
命中 → 标记「疑似重复待确认」，交人工，不自动写入。

## 3. 加拿大税务坑（重点）

- Intuit **sandbox 默认美国公司**，税制是 US Sales Tax，**没有 GST/HST/PST**。
- 加拿大 QBO 的税是 Automated Sales Tax，Bill 上通过行级 `TaxCodeRef` + `GlobalTaxCalculation` 表达；税码 Id 每个公司不同，要先 `SELECT * FROM TaxCode` 拉取。
- **验证策略**：核心链（识别→分类→建 Bill）先用美国 sandbox 跑通「不含税」骨架；加拿大税逻辑用一个加拿大区测试公司或生产灰度单独验证。→ 不阻塞 MVP 骨架，但税不能想当然。

## 4. 生产准入

- QBO API 免费、无按调用量计费。
- 生产密钥需过 **Intuit 安全审查（约 2–4 周）**：填 app 信息、隐私政策、数据用途、回调 URL 等。
- **行动**：MVP 第一天就注册开发者账号建 app 拿 sandbox 凭据；同时把生产审查材料准备起来并行提交。

## 5. 需要你提供的（凭据到位我就能实测）

1. 在 https://developer.intuit.com 注册开发者账号，新建一个 app（选 QuickBooks Online Accounting scope）
2. 从 app 的 **Keys & OAuth** 页拿 **Sandbox** 的 `Client ID` + `Client Secret`
3. Intuit 会自动给一个 **sandbox company**（美国）；记下授权后回调里的 `realmId`
4. 跑一次 OAuth 授权拿到 `refresh_token`（用官方 OAuth Playground 最快）
5. 把 `CLIENT_ID / CLIENT_SECRET / REFRESH_TOKEN / REALM_ID` 填进第 6 节脚本的环境变量

## 6. 可运行验证脚本（凭据到位后存为 `verify-qbo.mjs` 运行）

> Node 18+；`npm i intuit-oauth`。用 refresh token 换 access token，然后跑「读科目表 → 查/建供应商 → 建 Bill → 挂附件」全链。

```js
import OAuthClient from 'intuit-oauth';

const { CLIENT_ID, CLIENT_SECRET, REFRESH_TOKEN, REALM_ID } = process.env;
const BASE = 'https://sandbox-quickbooks.api.intuit.com';
const MV = '73'; // 待实测确认当前 minorversion

const oauth = new OAuthClient({ clientId: CLIENT_ID, clientSecret: CLIENT_SECRET, environment: 'sandbox', redirectUri: 'http://localhost' });
const { token } = await oauth.refreshUsingToken(REFRESH_TOKEN);
const AT = token.access_token;

const H = { Authorization: `Bearer ${AT}`, Accept: 'application/json', 'Content-Type': 'application/json' };
const q = async (sql) => (await fetch(`${BASE}/v3/company/${REALM_ID}/query?query=${encodeURIComponent(sql)}&minorversion=${MV}`, { headers: H })).json();

// 1. 读费用科目
const accts = await q("SELECT * FROM Account WHERE AccountType = 'Expense'");
const acctId = accts.QueryResponse.Account?.[0]?.Id;
console.log('费用科目样例:', accts.QueryResponse.Account?.[0]?.Name, acctId);

// 2. 查/建供应商
let v = await q("SELECT * FROM Vendor WHERE DisplayName = 'Claude Test Vendor'");
let vendorId = v.QueryResponse.Vendor?.[0]?.Id;
if (!vendorId) {
  const r = await fetch(`${BASE}/v3/company/${REALM_ID}/vendor?minorversion=${MV}`, { method: 'POST', headers: H, body: JSON.stringify({ DisplayName: 'Claude Test Vendor' }) });
  vendorId = (await r.json()).Vendor.Id;
}
console.log('供应商 Id:', vendorId);

// 3. 建 Bill
const billRes = await fetch(`${BASE}/v3/company/${REALM_ID}/bill?minorversion=${MV}`, {
  method: 'POST', headers: H,
  body: JSON.stringify({
    VendorRef: { value: vendorId },
    DocNumber: 'INV-VERIFY-001',
    TxnDate: '2026-07-07',
    Line: [{ DetailType: 'AccountBasedExpenseLineDetail', Amount: 42.5, AccountBasedExpenseLineDetail: { AccountRef: { value: acctId } } }],
  }),
});
const bill = (await billRes.json()).Bill;
console.log('✅ Bill 已创建 Id:', bill?.Id, 'TotalAmt:', bill?.TotalAmt);

// 4. 去重校验（应查到刚建的）
const dup = await q(`SELECT * FROM Bill WHERE DocNumber = 'INV-VERIFY-001'`);
console.log('去重 query 命中数:', dup.QueryResponse.Bill?.length);

// 5. 附件（多层 multipart，示意；真正实现用 form-data 库拼 file_metadata_01 + file_content_01 并 POST /upload）
console.log('附件上传：POST /v3/company/%s/upload（见文档动作4）', REALM_ID);
```

## 7. 对 MVP 的影响

- ✅ 录入链无阻塞，可以放心把它作为核心链终点。
- 🔧 建 app 拿 sandbox 凭据、准备生产审查材料 → 列入第一天行动项。
- 🔧 加拿大税作为「核心链骨架之后紧接的第一个增量」，不混进骨架验证。
