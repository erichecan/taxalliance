# QuickBooks Online 开发者账号 + 沙箱 获取指引

> 日期：2026-07-09
> 目的：从零拿到 QBO **sandbox 凭据**（Client ID / Secret / refresh token / realmId），填进 `.env`，我就能接「同步科目表 + 录入 Bill」。
> 事实来源：Intuit 官方文档（2026-07-09 核实，见文末 Sources）+ 本项目 [QBO 录入链验证](20260707-qbo-api-录入链验证.md)。

---

## 0. 先说清楚：你不需要什么

- ❌ 不用是会计、不用有真实客户、不用真实 QBO 订阅。
- ✅ Intuit 免费给你一个 **sandbox 沙箱公司**（一套假的 QBO 公司，自带默认科目表 + 供应商），整套对接都能在沙箱里开发测试。
- ⚠️ sandbox 默认是**美国公司**（没有 GST/HST）——先用它跑通「读科目 / 建供应商 / 建 Bill / 挂附件」的**不含税骨架**；加拿大税逻辑之后单独验。

---

## 1. 注册 Intuit 开发者账号

1. 打开 **<https://developer.intuit.com>** → 右上 **Sign up**（有 Intuit 账号直接 Sign in）。
2. 用邮箱注册一个 Intuit 账号，验证邮箱，登录进开发者门户。

## 2. 创建一个 App

1. 进 **Dashboard**（我的应用）→ **Create an app**。
2. 平台选 **QuickBooks Online and Payments**。
3. 勾选 scope：**`com.intuit.quickbooks.accounting`**（会计接口，读科目/建 Bill 都靠它）。
4. 给 app 起个名（如 `Easetax Dev`）。

## 3. 拿 sandbox 凭据 + 设 Redirect URI

1. 在 app 里进 **Development** 区 → **Keys & OAuth**（开发/沙箱密钥）。
2. 复制两个值：
   - **Client ID**
   - **Client Secret**
   > 注意区分：**Development**（沙箱，现在用）vs **Production**（生产，要过安全审查才有）。现在只拿 Development 的。
3. 在同一 **Keys & OAuth** 页，**Redirect URIs** 加两个：
   - `https://developer.intuit.com/v2/OAuth2Playground/RedirectUrl` ← 第 5 步用 Playground 拿 token 必需
   - `http://localhost:3000/api/clients/callback` ← 以后我们 app 自己的 OAuth 回调（先加上）

## 4. 沙箱公司（自动就有）

- 注册开发者账号时 Intuit 会**自动给一个 sandbox 公司**（美国）。
- 在 **<https://developer.intuit.com/v2/ui#/sandbox>** 能看到所有沙箱公司**及它们的 `realmId`**（realmId = 该账套的公司 ID，建 Bill 时要用）。
- 需要的话可在该页 **Add** 再建、或 reset 沙箱。

## 5. 用 OAuth 2.0 Playground 拿 refresh token + realmId（最快）

1. 打开 **OAuth 2.0 Playground**：<https://developer.intuit.com/app/developer/qbo/docs/develop/authentication-and-authorization/oauth-2.0-playground>（也能从 app dashboard 直接进）。
2. 选中你的 app（自动带出 Client ID/Secret），scope 勾 `com.intuit.quickbooks.accounting`。
3. 点授权 → 选你的 **sandbox 公司** → 同意。
4. Playground 会给你：
   - **Authorization Code** →（点下一步）换出 **Access Token** + **Refresh Token**
   - 回调 URL 里带 **realmId**（也可在第 4 步的 sandbox 页查到）
5. 记下 **Refresh Token**（约 100 天有效、每次刷新轮换）和 **realmId**。

## 6. 填进 `.env`

编辑 `apps/web/.env`（已 gitignore，勿提交）：

```bash
QBO_CLIENT_ID="你的 Client ID"
QBO_CLIENT_SECRET="你的 Client Secret"
QBO_ENVIRONMENT="sandbox"
QBO_REDIRECT_URI="http://localhost:3000/api/clients/callback"
# 初次用 Playground 拿到的，用于脚本直连沙箱验证：
QBO_REFRESH_TOKEN="Playground 给的 refresh token"
QBO_REALM_ID="沙箱公司的 realmId"
```

发我或自己填好告诉我即可（`QBO_REFRESH_TOKEN` / `QBO_REALM_ID` 我会补进 `.env.example` 模板）。

## 7. 关键坑（提前知道）

| 坑 | 说明 |
|----|------|
| **沙箱是美国公司** | 没有 GST/HST。先跑不含税骨架；加拿大税用加拿大区测试公司或生产灰度另验 |
| **生产准入要审查** | 生产密钥需过 Intuit 安全审查（约 2–4 周）。开发全程用 sandbox，审查材料并行准备 |
| **refresh token 会轮换** | 每次刷新返回新的 refresh token，要存最新的（我们已按契约 G5 加密存 `Client.qboRefreshToken`） |
| **限流** | 约 500 请求/分钟/公司，超限 429，需退避 |

## 8. 凭据到位后我会做什么

1. **同步科目表**：`SELECT * FROM Account` → 写入 `GlAccountCache`，把现在 seed 的 17 个演示科目换成沙箱真实科目表（分类的靶子从此是真的）。
2. **录入链**：查/建 Vendor → 建 Bill（行 `AccountRef`=分类结果）→ 挂原件附件 → 去重 → 状态 `synced`。
3. 先用 [验证脚本](20260707-qbo-api-录入链验证.md#6-可运行验证脚本凭据到位后存为-verify-qbomjs-运行) 直连沙箱跑通五个动作，再接进 app。
