## 二、这次讨论定位到的核心问题（待解，等 key）

**多列表格的明细，Veryfi 结构化时被压成了"描述 + 一个 total"。**

- 例：车行发票每行有 principal / misc / interest / fees / total 多列，Veryfi 的 `line_items` 只给了描述 + Total 列，**principal/interest/fees 没结构化出来**。
- **但 `ocr_text`（Veryfi 返回的全文，存在 `Extraction.rawJson.ocr_text`）里所有列的文字都在**，一个数字没漏。
- **解法**：`ocr_text → Claude` 补全——Veryfi 固定行模型搞不定的多列/异形格式，交给 Claude 从全文解析出全部列。这对记账有实际意义（利息进利息支出、费用进融资费，列拆不开科目就分不对）。
