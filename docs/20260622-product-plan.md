# Taxalliance 产品规划文档

> 整理日期：2026-06-22  
> 基于与 Claude Code 的深度调研与讨论，汇总两大业务方向的功能规划、技术选型与实施路径。

---

## 一、业务方向概览

### 方向 A：T2 AI 企业报税工具（SaaS）

面向加拿大小型企业和会计师事务所，提供从文件收集、AI 识别、科目映射、T2 表格自动填写，到 CRA 电子提交的端到端流程。

### 方向 B：税务联盟加盟品牌

建立税务机构连锁加盟体系，加盟商缴纳加盟费后使用统一品牌，在 Google Maps 展示连锁标志，以 A 的 AI 工具作为核心差异化竞争优势向加盟商赋能。

**最终目标**：A 和 B 合并，用 AI 工具平台赋能加盟商，形成"技术驱动的税务联盟"。

## 二、T2 AI 报税工具 — 功能模块详解

### 模块 1：客户门户（Client Portal）

客户侧入口，会计师将链接发给客户，客户在此上传报税材料。

- 客户账户注册 / 登录（邮件验证）
- **定时提醒系统**：按报税截止日倒计时自动发送邮件/短信（如"距 T2 截止还有 30 天，请上传以下材料"）
- 材料清单模板：会计师预设所需文件类型（银行流水、收据、HST 报表等）
- **多格式上传**：PDF、图片（扫描件/手机拍照）、CSV、Excel，支持批量上传
- 上传进度追踪：客户看到"已上传 3/6 项，缺少：2024 年 Q4 银行流水"
- 文件存储：加拿大境内服务器（PIPEDA 合规要求，推荐 AWS 加拿大区或 Azure 加拿大）

---

### 模块 2：AI 文档识别层（OCR Engine）

**技术选型：Veryfi API**（详见第三节选型对比）

- 调用 Veryfi REST API，秒级返回结构化 JSON
- 支持文件类型：PDF、JPEG/PNG（扫描件）、CSV、Excel、HEIC
- 识别内容：
  - **收据 / 发票**：金额、日期、供应商名、GST/HST 税额、行项目
  - **银行流水 PDF**：逐行提取每笔交易（日期、描述、金额、余额）
  - **CSV / Excel 银行流水**：直接解析结构化数据
- 识别置信度评分：低于阈值标记为"需人工确认"
- 重复检测：同一文件上传两次自动提示

---

### 模块 3：GIFI 科目映射引擎（核心差异化）

**市场最大空白**，现有工具均为手动或简单规则映射，本系统用 AI 实现。

- **输入**：Veryfi 识别出的每条交易（描述 + 金额 + 供应商）
- **输出**：对应的 GIFI 代码（如"Tim Hortons $12.50" → GIFI 8811 餐饮费）
- **实现方式**：LLM（Claude API）+ 规则引擎混合
  - 规则引擎：处理明确映射（银行费用 → 8710，薪酬 → 9060）
  - LLM：处理模糊描述（如"Amazon"结合上下文判断是办公用品还是 IT 费用）
- **学习机制**：会计师修正后记录，用于下次同类交易自动映射
- **置信度分级**：
  - 高：自动通过
  - 中：AI 建议 + 人工确认
  - 低：必须人工处理

---

### 模块 4：T2 表格预填（Form Auto-Population）

将映射好的数据自动填入 T2 对应附表：

| 附表 | 内容 | 数据来源 |
|------|------|----------|
| Schedule 100 | 资产负债表（GIFI） | 会计软件导出 |
| Schedule 125 | 损益表（GIFI） | 映射后的收支数据 |
| Schedule 140 | 简版损益表 | Schedule 125 汇总 |
| Schedule 1 | 净收入调整（会计利润→税务利润）| 自动计算 |
| Schedule 7 | 小企业税率申请 | 基于营收判断资格 |
| Schedule 8 | CCA 资本成本摊销 | 固定资产清单 |
| T2 主表 | 基本信息 + 税额计算 | 上述附表汇总 |

---

### 模块 5：会计师复核工作台（Review Workbench）

质量控制核心界面，所有 AI 结果必须经会计师确认后才能提交。

- **三栏布局**：左栏原始文件预览 | 中栏 AI 识别结果 | 右栏 T2 表格预览
- 逐条复核：每条映射显示来源文件、识别内容、建议 GIFI 代码、置信度
- 内联编辑：直接修改任何字段，实时同步更新 T2 表格
- 异常提示：金额超历史均值 3 倍、科目不寻常自动高亮
- 审核状态：未审核 / 审核中 / 已确认 / 已退回客户补充
- 审计追踪：记录每次修改的操作人和时间

---

### 模块 6：CRA EFILE 提交（Filing Submission）

**技术选型：TaxCycle 对接**（详见第四节）

**架构**：

```
你的系统生成 GIFI CSV 文件
    ↓
[可选] 通过 TaxCycle API 自动导入（需签协议）
[或]   导出文件给会计师手动导入 TaxCycle
    ↓
TaxCycle 自动填充 Schedule 100 / 125 / 140
    ↓
会计师在 TaxCycle 复核 + 手动点击"Transmit Now!"提交 CRA
```

**重要限制**：TaxCycle 的 CRA EFILE 提交无法通过 API 远程触发，会计师必须在桌面端手动完成最后一步。这是现阶段的技术边界，Phase 3 自申请 CRA EFILE 认证后可彻底自动化。

---

### 模块 7：报告与分析（Reports & Analytics）

**给客户的报告**（非技术语言）：
- 本年度应缴税款估算
- 主要收支分类汇总（饼图 / 柱图）
- 与上年度对比
- 下一步行动项（如"需补缴 $3,200，截止 3 月 31 日"）

**给会计师的 Dashboard**：
- 客户列表 + 每户处理状态（未开始 / 进行中 / 等待客户 / 已提交）
- AI 映射准确率统计（自动通过 vs 人工修改比例）
- 批量截止日提醒

## 三、税务联盟品牌 — 功能模块详解

### 模块 8：加盟商管理后台

- 在线加盟申请表 + 资质审核流程
- 合同电子签署（DocuSign 或 TaxFolder）
- 加盟商档案：联系人、地址、许可区域、加入日期
- 费用管理：加盟费收取记录、月度特许费计算（按营收比例）
- 培训材料库：操作手册、税务知识、系统使用教程

### 模块 9：Google Maps 品牌管理系统

- 批量创建 Google Business Profile（GBP）档案
- 统一管理 NAP（名称/地址/电话），总部掌控品牌字段
- 向加盟商授权"位置管理员"权限（仅可改营业时间和本地图片）
- 评论监控：聚合所有门店 Google 评论，统一回复规范
- 报税季内容日历：1 月统一推送"春季报税优惠"GBP 帖文

### 模块 10：加盟商技术平台入口

- 每个加盟商获得独立的 T2 AI 工具账户（白标，显示联盟品牌）
- 按加盟等级限制客户数（如 50 客户/年 或无限）
- 使用数据看板：各加盟商系统使用率、提交数量

### 模块 11：品牌资产中心

- Logo、名片模板、门店设计规范下载
- 营销素材：海报、社交媒体模板
- 品牌锁定分层：可定制部分（加盟商名字）和锁定部分（品牌元素）分开管理

## 四、OCR 技术选型：Veryfi vs Dext

### 结论：选 Veryfi

Dext Prepare（OCR 产品）在英语市场**没有公开 API**，无从集成。Veryfi 是开发者优先的 API 基础设施层，完全符合需求。

### 详细对比

| 维度 | Veryfi | Dext |
|------|--------|------|
| **公开开发者 API** | ✅ 完全开放 REST API | ❌ Dext Prepare 无公开 API（英语市场）|
| **银行流水逐行提取** | ✅ 每笔交易含描述、分类、金额、余额 | ⚠️ 支持但不通过 API 暴露，有月度限额 |
| **处理速度** | **秒级**（同步 API）| 典型约 30 分钟（异步队列）|
| **Webhook** | ✅ 支持异步回调 | ❌ 无原生 Webhook |
| **SDK** | 18 种语言，全部开源 | 无正式 SDK |
| **整体准确率** | 99%+（银行流水 99.8%）| 97.5% 全文档 / 99.5% 字段级 |
| **SLA Uptime** | **99.995%** | 99.99%（声称）|
| **人工干预** | ❌ 纯 AI 管道，数据不流经人手 | ⚠️ HITL 模式，有人工复核 |
| **GL Code 映射** | ✅ 内置 + 完全可定制 | ✅ 连接 Xero/QBO 后支持 |
| **PIPEDA 合规** | ✅ | ✅ |
| **GST/HST 字段** | ✅ 支持 CAD，91+ 种货币 | ✅ 支持，含 PST/QST |
| **定价透明度** | ✅ 官网公开价格表 | ❌ 需联系销售 |

## 五、TaxCycle 对接方式

### API 状态

TaxCycle 有 Software Developer API License（2025 年 1 月更新），但**无公开自助注册**，需通过以下方式申请：
- 邮件：info@taxcycle.com
- 电话：1-888-841-3040

### 数据导入路径

系统生成符合 GIFI 规范的 CSV 文件，TaxCycle 支持以下格式导入：
- `.GFI`、`.CSV`、`.TXT`
- 导入后自动填充 Schedule 100、125、140

### EFILE 提交限制

| 环节 | 能否自动化 | 说明 |
|------|-----------|------|
| 财务数据导入 TaxCycle | ✅ 可自动化 | 生成 GIFI CSV，通过 API 或手动导入 |
| T2 表格填充 | ✅ 自动完成 | 导入后自动填充附表 |
| CRA EFILE 提交 | ❌ **无法自动化** | 会计师必须在 TaxCycle 桌面端手动点击提交 |

### 会计师所需资质

提交 T2 需具备以下之一：
- **EFILE 账号**：向 CRA 申请，通过适格性审查（约 30 天），每年续期
- **Web Access Code (WAC)**：企业向 CRA 申请，仅对特定税年有效

## 六、分阶段建设路线图

### Phase 1（0–6 个月）：文件收集 + OCR 识别

**目标**：会计师能用系统收材料，Veryfi 完成 OCR，验证市场需求。

- 模块 1：客户门户（上传、提醒）
- 模块 2：Veryfi API 集成（OCR 识别）
- 模块 7：基础报告（识别结果展示）
- **无需 CRA 认证**，快速上线验证

**技术栈**：Next.js App Router + PostgreSQL + Veryfi API + AWS 加拿大区存储

---

### Phase 2（6–12 个月）：GIFI 映射 + T2 预填 + 复核工作台

**目标**：核心差异化功能上线，完成完整 AI 报税流程。

- 模块 3：GIFI 科目映射引擎（Claude API + 规则引擎）
- 模块 4：T2 表格预填（Schedule 100/125/140/1/7/8）
- 模块 5：会计师复核工作台
- 同步联系 TaxCycle 申请 API 合作协议

---

### Phase 3（12–18 个月）：TaxCycle 对接 + 完整提交流程

**目标**：打通从数据识别到 CRA 提交的完整链路。

- 模块 6：TaxCycle GIFI CSV 导出 + API 对接
- 完整 T2 附表支持
- 同步申请 CRA 自有 EFILE 认证（认证周期约 1 年，提前启动）

---

### Phase 4（18+ 个月）：联盟品牌体系

**目标**：以 AI 工具为核心竞争力，建立加盟商生态。

- 模块 8–11：加盟商管理、GBP 品牌系统、白标技术平台、品牌资产中心
- Google Maps 体系化运营
- 加盟商培训与支持体系

## 七、关键风险与应对

| 风险 | 严重程度 | 应对策略 |
|------|---------|---------|
| TaxCycle EFILE 最终提交无法自动化 | 中 | 接受现阶段由会计师手动完成，作为"人工复核"产品优势；Phase 3 自申请 CRA 认证后解决 |
| AI GIFI 映射错误导致申报错误 | 高 | 所有 AI 映射必须经会计师人工复核确认，不允许全自动提交；建立错误责任说明 |
| Veryfi $500/月最低消费 | 低 | 初期用 100 份/月免费额度开发，上线前评估实际用量 |
| CRA EFILE 自有认证周期长（约 1 年）| 中高 | Phase 1-2 通过 TaxCycle 绕过，Phase 3 再自行申请 |
| 客户数据隐私（PIPEDA）| 高 | 数据存储加拿大境内；SOC 2 Type II 认证；明确数据处理协议 |
| 魁北克省 CO-17 单独提交体系 | 中 | 初期聚焦联邦 T2，魁北克支持作为后续扩展 |
